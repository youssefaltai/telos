# Agentix

> A minimal framework for fully recursive, intent-driven agentic development.
> You define the goal. Agents define everything else.

---

## Philosophy

The boundary between human work and agent work is not at the task level. It is at the *intent* level.

You provide one thing: **what matters and why.**
The system provides everything beneath that — including how to organize itself to achieve it.

---

## How It Works

```
You write INTENT.md
    └── Bootstrap Meta-Agent reads it
          └── Designs the optimal agent topology for your goal
                └── Writes agent definitions to /agents/
                      └── Orchestrator coordinates agents toward the goal
                            └── Specialist agents do the work
                                  └── Critic agents validate the work
                                        └── Prompt Engineer improves the agents
                                              └── Memory Agent curates what's learned
                                                    └── System gets better over time
```

Every layer is agent-owned. Only the top is human-owned.

---

## Getting Started

### 1. Define your intent

Edit `INTENT.md`. Be as high-level as you want. The agents will ask if they need more.

### 2. Run the bootstrap

In Claude Code (or any Claude interface):

```
Read CLAUDE.md and begin.
```

The Bootstrap Meta-Agent will design your agent topology automatically.

### 3. Give the first task

Once bootstrapped, talk to the Orchestrator:

```
[Your task or goal here]
```

The Orchestrator routes to the right agents. You only get interrupted when it genuinely needs you.

## Directory Structure

```
agentix/
├── INTENT.md          # YOU write this. Agents read it. No one modifies it.
├── CONSTITUTION.md    # Invariant rules. Applies to every agent at every level.
├── CLAUDE.md          # Entry point for Claude Code. First thing Claude reads.
│
├── bootstrap/
│   └── meta-agent.md  # The agent that designs all other agents
│
├── agents/
│   ├── core.md        # Always-present agents: Orchestrator, Critic, Memory, Prompt Engineer
│   └── [...]          # Project-specific agents, generated by the Meta-Agent
│
├── memory/
│   ├── state.md       # Current system state
│   └── [...]          # Curated persistent knowledge
│
├── checkpoints/
│   └── [...]          # Timestamped snapshots before every significant action
│
└── logs/
    └── [...]          # Raw agent outputs, flagged for Memory Agent curation
```

## Core Principles

**Minimal surface area** — The framework is ~5 files. Everything else is generated.

**Locked intent** — INTENT.md cannot be modified by any agent. It is the root truth.

**Separation of concerns** — No agent produces and validates its own output.

**Reversibility** — Every action is preceded by a checkpoint. The system can always go back.

**Tool agnosticism** — Claude Code is the default runtime. It is not the only option.

**Recursive by design** — Agents can spawn sub-agents. Meta-agents can redesign agents. The recursion goes as deep as the problem requires.

---

## The One Thing That Stays Human

In a fully recursive system, one input is irreducible: **does this matter?**

You answer that in INTENT.md.
Everything else is agent territory.

---

## Contributing / Extending

The framework is intentionally minimal. Domain-specific extensions (e.g., an agent topology for web development, data pipelines, or research) can be added as presets in `/bootstrap/presets/`.

The Meta-Agent can be given a preset to start from rather than designing from scratch.
